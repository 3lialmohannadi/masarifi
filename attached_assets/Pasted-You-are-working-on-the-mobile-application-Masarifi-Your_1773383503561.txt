You are working on the mobile application Masarifi.



Your task is to implement a new feature called:





Debts Management (المديونيات)





This feature must be built in a clean, scalable, maintainable way so it can grow later without needing to be rebuilt.









Important rules





Do NOT redesign the application.
Do NOT change the current color palette, branding, or UI identity.
The new feature must match the existing Masarifi design system, components, spacing, typography, and navigation style.
Keep the UX simple, clean, and mobile-friendly.
Build the feature in a modular and scalable architecture.
Make sure the code is reusable and future-proof.










Feature location





The Debts section must NOT be placed in the main bottom tabs.



It must be accessible from:





More → Debts





المزيد → المديونيات









Main feature goal





The Debts section should allow users to:



Add debts
View debts
Edit debts
Delete debts
Register payments
Track repayment progress
Filter debts
View simple debt statistics




This is the first version (MVP), but it must be built in a way that supports future expansion.









Architecture requirements





Implement the Debts feature using a clean structure.



Please ensure the following:



Separate UI components from business logic
Use reusable components where possible
Keep form logic organized
Keep debt calculation logic centralized
Make the feature easy to extend later for:
notifications
due reminders
advanced analytics
integrations
export/reporting





If needed, create a dedicated folder/module for debts.



Suggested structure example:

debts/services




Use whatever matches the app’s existing architecture, but keep it clean and modular.

Validation requirements





Implement proper validation:

Entity / person name is required

Make validation user-friendly and clear.









Payment management





Inside each debt, users must be able to:





1. Pay installment





For installment-based debts.





2. Partial payment





Allow entering a custom payment amount.



This must:



reduce the remaining amount
increase paid amount
update repayment progress






3. Full settlement





This must:



set remaining amount to zero
mark status as fully paid
update repayment progress to 100%




If useful, keep a payment history structure for future use.
